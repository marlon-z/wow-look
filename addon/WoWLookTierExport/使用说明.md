# WoWLookTierExport 使用说明

这个插件用于导出当前赛季职业套装数据，输出内容包括：

- 13 个职业的 65 件套装
- 每件套装的英雄轨道展示 tooltip
- 套装名、5 件成员、部位、护甲类型、主副属性
- 按专精展开的 `2 件 / 4 件` 套装效果

## 命令

```text
/wowtierexport all
/wowtierexport monk
/wowtierexport monk,druid,mage
/wowtierexport summary
/wowtierexport help
```

## 导出结果

导出后会写入：

```text
WTF/Account/<账号>/SavedVariables/WoWLookTierExport.lua
```

核心字段：

- `summary`
- `classes`
- `payload`

其中：

- `classes` 是 Lua 表，适合直接调试查看
- `payload` 是 JSON 字符串，适合后续 Node 脚本解析

## 数据边界

这版插件当前使用统一的英雄赛季展示 link：

- 难度：英雄
- 轨道：英雄 `2/6`

所以导出的是真实可渲染的英雄展示版本，不是玩家个人升级后的实例值。

## 复用说明

这个插件可以复用到下一赛季，但要区分：

- 可复用的是导出框架
- 需要替换的是赛季配置

### 可复用的部分

- 构造赛季展示 link
- 读取 tooltip
- 解析装等、部位、护甲类型、主副属性
- 解析套装名、5 件成员
- 按专精导出 `2 件 / 4 件` 效果
- 导出 `summary / classes / payload`

### 下个赛季需要更新的部分

1. `TIER_SETS`
- 13 职业新的套装 `itemId`
- 新的套装名

2. 赛季 link 参数
- `SEASON_BONUS_ID`
- `RAID_HEROIC_DIFFICULTY`
- `RAID_HEROIC_CONTEXT`

也就是说，这个插件不是“永久零维护通用插件”，而是：

**一个可以跨赛季复用的套装导出框架。**

换赛季时通常只需要更新：

- 一份职业套装配置
- 一组赛季展示参数

主逻辑不需要重写。

## WoWLookTierProbe 的定位

`WoWLookTierProbe` 是前期实验插件，主要用于验证：

- 裸 `itemId` 是否能拿到真实套装属性
- 外观套装聊天 link 是否可作为真值来源
- 专精效果能否按 `specId` 展开

这些问题已经在当前方案里验证完成。

对当前项目主线来说：

- 正式采集插件：`WoWLookTierExport`
- 实验验证插件：`WoWLookTierProbe`

如果你只保留正式链路，`WoWLookTierProbe` 已经没有继续安装和使用的必要。
